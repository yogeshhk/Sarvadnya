### 2. `ask_finance/agentic_rag_system.py` — Incomplete implementation marked

- Added `TODO` block in module docstring explaining which methods are missing
- `_build_graph()` now raises `NotImplementedError` immediately rather than silently failing when the truncated line is reached at runtime
- `ask_finance/README.md` updated with an implementation status table at the top

### 3. `ask_almanack/streamlit_main.py` — Hardcoded API key placeholder fixed

Before: `os.getenv("GROQ_API_KEY", "your_groq_api_key_here")` — passes a literal placeholder string to the Groq client, causing a confusing authentication error.

After: Explicit check with `st.error()` + `st.stop()` before the client is constructed. Users see a clear "GROQ_API_KEY not set" message instead of an API auth error from inside the library.

### 4. `CitationQueryEngine` unified into shared module

Before: `CitationQueryEngine`, `CitationConversationEngine`, and `extract_text_from_node()` were copy-pasted verbatim in both:
- `ask_yogasutra/graphrag/graphrag_backend.py`
- `ask_yogasutra/linearrag/linearrag_backend.py`

After: All three live in `ask_yogasutra/utils/citation.py` with added docstrings explaining *why* only three Sanskrit fields are indexed. Both backends import from the shared module. ~120 lines of duplicate code eliminated.

### 5. Stale commented-out import blocks removed

Both `graphrag_backend.py` and `linearrag_backend.py` carried 10–13 lines of commented-out old-style llama_index imports (pre-0.10 migration artifacts). These were removed. Also removed:
- Duplicate `import os` in `linearrag_backend.py`
- Stale commented-out `check_model_path()` method body in `linearrag_backend.py`
- Stale `# from llama_index.llms.llama_cpp import LlamaCPP` comment in `graphrag_backend.py`

### 6. `ask_faqs_chatbot/faqengine.py` — Exception logging improved

Before: `except Exception as e: print(e)` — only printed the exception message string, discarding the traceback and exception type entirely.

After: `logger.exception("Query failed for input: %r", usr)` — logs full traceback at ERROR level via the standard `logging` module. A module-level `logger = logging.getLogger(__name__)` was added.
